What is it ?

Pyros is a software layer on top of the ROS framework, allowing you to interface your ROS systems with : - standard python dev dynamic workflow (no generation, compilation time wasted) - standard python packages, potentially leveraging NNNNN existing packages on PyPI - any existing Cloud REST Services (via HTTP) - any existing Frontend Application and GUI (via HTTP) - any existing IoT infrastructure (via MQTT, RIOT, etc.)

What can I do with it ?

Pyros is a distributed framework, relying on standard protocols, so you can can come up with any distributed architecture that fit your needs : - Robots accessing REST micro services on the cloud (like http://gocart.yujinrobot.com/ ) - Controling and Monitoring remote robots (like https://flytbase.com/ ) - Do your robot application development in a standard dynamic IDE (like PyCharm) - Controling a robot from a webapp on your mobile phone - and much more !
